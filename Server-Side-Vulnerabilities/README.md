# Server-Side Vulnerabilities

This section documents my hands-on exploitation of server-side vulnerabilities using labs from PortSwigger Web Security Academy.

The focus is on understanding how real-world applications break and how attackers exploit backend logic to gain unauthorized access, execute code, and extract sensitive data.

---

## 🔥 Skills Demonstrated

- Exploiting server-side vulnerabilities in real scenarios
- Using Burp Suite for intercepting and modifying requests
- Identifying injection points and bypassing input validation
- Performing Remote Code Execution (RCE)
- Analyzing application behavior through responses
- Understanding backend logic and attack surface

---

## 📚 Topics Covered

- Path Traversal
- Access Control Vulnerabilities
- Authentication Vulnerabilities
- SQL Injection (SQLi)
- Server-Side Request Forgery (SSRF)
- File Upload Vulnerabilities (RCE)
- OS Command Injection

---

## ⚙️ Tools Used

- Burp Suite (Proxy, Repeater, Intruder)
- Browser (manual testing)
- Kali Linux environment

---

## 🧠 Key Concepts Learned

- How improper input validation leads to critical vulnerabilities
- Exploitation chains (e.g., SSRF → internal access → admin actions)
- Bypassing security mechanisms (MIME type, authentication, access control)
- Command execution on server via injection vulnerabilities
- Importance of secure coding practices (parameterized queries, validation)
